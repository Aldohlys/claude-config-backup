# Build Package Command

Automated R package build and deployment workflow for the R trading system.

**INSTRUCTIONS FOR CLAUDE:**

You MUST follow these steps when executing this command:

## Step 1: Parse Arguments

Parse the package name and any flags from $ARGUMENTS:
- Package name is always the first argument
- Check for `auto` flag (skips user prompt, auto-increments patch)
- Check for `auto minor` or `auto major` (auto-increments specified version)
- Check for `--no-deploy` flag (build only, no deployment)

## Step 2: Read Current Version

Before prompting the user, read the package's DESCRIPTION file to show them the current version.

Example:
```bash
cd C:/Users/aldoh/Documents/RApplication/<package_name> && cat DESCRIPTION | grep "^Version:"
```

## Step 3: Ask User About Version Increment (MANDATORY unless 'auto' flag present)

**MANDATORY**: If the `auto` flag is NOT present, you MUST ask the user:

"Current version: X.Y.Z

Would you like to increment the version?
- **n** or **no** - Keep current version (no increment)
- **patch** - Bug fixes (X.Y.Z → X.Y.Z+1)
- **minor** - New features (X.Y.Z → X.Y+1.0)
- **major** - Breaking changes (X.Y.Z → X+1.0.0)

Your choice:"

Wait for user response and parse it.

## Step 3b: Generate Detailed Change Summary (will be done by build_package.R)

The build_package.R script will automatically:
1. Read the latest CHANGELOG.md entry for the current version
2. Analyze git diff to identify changed files and functions
3. Generate a DETAILED commit message with:
   - **Title**: "Build <package> v<version>: <concise summary of main change>"
   - **Body**: Bullet points describing what/how/why/impact
4. Required format example:
   ```
   Build Tdata v5.7.28: Fix intraday option data retrieval and add exchange support

   - Fixed on_demand_historical.py: what_to_show parameter was hardcoded to TRADES
   - Data collection succeeded but retrieval failed with wrong what_to_show filter
   - Updated _fetch_single_contract_data() to accept what_to_show parameter
   - Added exchange parameter support throughout metadata chain
   - RPreTrade auto-retrieves currency, multiplier, exchange from Ticker table
   - Fixes: 'No data available' warning despite successful data collection
   ```
5. **Avoid generic messages** like "package maintenance", "version update"

No user input needed - Claude in build_package.R will handle this by reading CHANGELOG.md and analyzing git changes.

Based on the user's response or flags, execute the appropriate Rscript command:

**If 'auto' flag present (no user prompt needed):**

cd C:/Users/aldoh/Documents/RApplication && "/c/Program Files/R/R-4.4.3/bin/Rscript.exe" -e "source('build_package.R'); build_package('<package_name>', auto_version = TRUE, version_type = 'patch')"

**If 'auto minor' or 'auto major':**
